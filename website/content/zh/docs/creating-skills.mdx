# 创建技能

技能是可重用的指令，用于扩展你的 AI 助手的能力。本指南将向你展示如何创建自己的技能。

## 技能结构

每个技能是一个至少包含 `SKILL.md` 文件的文件夹：

```
skill-name/
├── SKILL.md              # 必需：技能定义
├── scripts/              # 可选：辅助脚本
│   └── helper.py
├── templates/            # 可选：文件模板
│   └── template.md
└── references/           # 可选：参考资料
    └── guide.md
```

## SKILL.md 格式

`SKILL.md` 文件定义你的技能：

```markdown
---
name: my-skill
description: 简要描述这个技能的功能和用途
---

# 我的技能

## 概述

解释这个技能的功能以及何时使用它。

## 指令

AI 需要遵循的分步指令。

### 步骤 1：分析

描述第一步...

### 步骤 2：执行

描述第二步...

## 示例

### 示例 1

展示一个输入示例和预期输出。

## 注意事项

任何额外的上下文或警告。
```

## 最佳实践

### 1. 清晰的指令

编写的指令应该：

- 具体且可操作
- 分解为编号步骤
- 在有帮助的地方包含示例

### 2. 适当的范围

每个技能应该：

- 专注做好一件事
- 可在项目间重用
- 不与其他技能重叠

### 3. 良好的命名

- 使用小写字母和连字符：`my-skill-name`
- 描述性但简洁
- 避免通用名称如 `helper` 或 `utils`

## 添加到工作流

1. 在 `.claude/skills/` 中创建你的技能文件夹：

```bash
mkdir -p .claude/skills/my-skill
```

2. 创建 `SKILL.md` 文件：

```bash
touch .claude/skills/my-skill/SKILL.md
```

3. 编辑 `SKILL.md` 写入你的技能定义

4. 用你的 AI 助手测试技能

## 示例技能

这是一个完整的文档生成技能示例：

```markdown
---
name: write-docs
description: 为代码生成文档
---

# 文档生成器

## 概述

为代码文件、函数和模块生成全面的文档。

## 指令

### 步骤 1：分析代码

- 识别代码的目的
- 记录所有公共函数和类
- 理解输入/输出类型

### 步骤 2：生成文档

- 编写模块级文档字符串
- 为每个公共函数编写文档
- 包含参数类型和返回值
- 添加使用示例

### 步骤 3：审查

- 确保文档准确
- 检查完整性
- 验证示例可用

## 输出格式

使用项目现有的文档风格。默认：

- JavaScript/TypeScript 使用 JSDoc
- Python 使用 Docstrings
- C# 使用 XML 注释
```
